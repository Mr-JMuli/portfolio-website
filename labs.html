Lab 1: Network Traffic Analysis with Wireshark

Problem Statement: Capture and analyze HTTP and FTP traffic from a simulated network to extract sensitive information.

Approach: Used filters in Wireshark to isolate relevant protocols, followed TCP streams, and extracted plain text credentials.

Tools Used: Wireshark, Kali Linux, VirtualBox

Screenshot:

Lessons Learned: Importance of encryption, use of filters (e.g. `http`, `ftp.request.command`), and how attackers can sniff credentials in unencrypted traffic.

Lab 2: DNS Query Tracing

Problem Statement: Understand and trace DNS query resolution from client to authoritative DNS server.

Approach: Used `dig`, Wireshark, and DNS debugging tools to observe resolution flow.

Tools Used: dig, Wireshark, nslookup, Kali Linux

Screenshot:

Lessons Learned: Role of each DNS server in resolution, how caching works, and how to detect DNS spoofing.

Lab 3: Basic Linux File Permissions

Problem Statement: Set correct file permissions and understand privilege escalation risks in Linux.

Approach: Used `chmod`, `chown`, and permission-based user testing to validate access.

Tools Used: Kali Linux, Bash, ls, chmod, chown

Screenshot:

Lessons Learned: Principle of least privilege, proper access control, and how improper permissions lead to vulnerabilities.
